# Decision Log

## 2026-06-16：采用建议技术栈

- 决策：空项目采用 Java 17 target、Spring Boot 3、Spring Security、MyBatis-Plus、MySQL、Flyway、Maven；前端采用 Vue 3、TypeScript、Vite、Element Plus、Pinia、Vue Router、Axios、ECharts。
- 原因：用户已给出明确技术栈建议，且适合长期全栈项目和 AI Agent 分阶段开发。

## 2026-06-16：先做文档体系和项目骨架

- 决策：第一轮先完成 Agent 文档体系，再进入 Phase 1 骨架。
- 原因：项目目标是长期自主开发，必须优先保证接手、状态、任务池和停止条件清晰。

## 2026-06-16：AI provider 先做 mock

- 决策：当前阶段不接真实 AI API，只保留 mock provider 方向。
- 原因：避免真实 API Key 和外部服务阻塞，符合安全边界。

## 2026-06-16：先做记账核心，不先做 OCR

- 决策：MVP 聚焦用户、空间、记账、购物、库存和基础统计，OCR 后期再做。
- 原因：记账和库存是生活数据闭环基础，OCR 依赖更复杂的外部能力和准确性验证。

## 2026-06-16：家庭空间放在 Phase 3

- 决策：用户鉴权后再实现家庭空间。
- 原因：空间权限依赖用户身份，先建立用户模型能降低后续数据隔离风险。

## 2026-06-16：当前不接支付

- 决策：不开发支付、银行卡绑定、自动下单。
- 原因：支付和自动购买属于高风险能力，超出当前生活记录和规划范围。

## 2026-06-16：用户表物理命名为 users

- 决策：逻辑用户表对应文档中的 `user`，物理表名使用 `users`。
- 原因：避免与数据库系统用户概念和潜在保留字冲突，降低 SQL 方言差异风险。

## 2026-06-16：JWT 使用本地 HMAC 实现

- 决策：P0-005 暂不引入额外 JWT 库，使用 Java 标准库实现 HS256 JWT 签发和校验。
- 原因：减少早期依赖面，同时保持 JWT 行为可测试；后续如需更完整 JWT/JWK 能力再引入成熟库。
