# Decision Log

## 2026-06-16：采用建议技术栈

- 决策：空项目采用 Java 17 target、Spring Boot 3、Spring Security、MyBatis-Plus、MySQL、Flyway、Maven；前端采用 Vue 3、TypeScript、Vite、Element Plus、Pinia、Vue Router、Axios、ECharts。
- 原因：用户已给出明确技术栈建议，且适合长期全栈项目和 AI Agent 分阶段开发。

## 2026-06-16：先做文档体系和项目骨架

- 决策：第一轮先完成 Agent 文档体系，再进入 Phase 1 骨架。
- 原因：项目目标是长期自主开发，必须优先保证接手、状态、任务池和停止条件清晰。

## 2026-06-16：AI provider 先做 mock

- 决策：当前阶段不接真实 AI API，只保留 mock provider 方向。
- 原因：避免真实 API Key 和外部服务阻塞，符合安全边界。

## 2026-06-16：先做记账核心，不先做 OCR

- 决策：MVP 聚焦用户、空间、记账、购物、库存和基础统计，OCR 后期再做。
- 原因：记账和库存是生活数据闭环基础，OCR 依赖更复杂的外部能力和准确性验证。

## 2026-06-16：家庭空间放在 Phase 3

- 决策：用户鉴权后再实现家庭空间。
- 原因：空间权限依赖用户身份，先建立用户模型能降低后续数据隔离风险。

## 2026-06-16：当前不接支付

- 决策：不开发支付、银行卡绑定、自动下单。
- 原因：支付和自动购买属于高风险能力，超出当前生活记录和规划范围。

## 2026-06-16：用户表物理命名为 users

- 决策：逻辑用户表对应文档中的 `user`，物理表名使用 `users`。
- 原因：避免与数据库系统用户概念和潜在保留字冲突，降低 SQL 方言差异风险。

## 2026-06-16：JWT 使用本地 HMAC 实现

- 决策：P0-005 暂不引入额外 JWT 库，使用 Java 标准库实现 HS256 JWT 签发和校验。
- 原因：减少早期依赖面，同时保持 JWT 行为可测试；后续如需更完整 JWT/JWK 能力再引入成熟库。

## 2026-06-16：新增项目内 Agent skills

- 决策：在 `agent-skills/` 中维护 LifePilot 项目专属 skills，首批包含自主开发、前端设计、本地 Web 验证和文档协作。
- 原因：项目已有长期 Agent 协作协议，skills 可以把高频、可复用、容易漏步骤的流程按任务加载，减少上下文负担；选择参考 Anthropic 示例而不是整包引入，避免不合技术栈或许可证不清晰的内容进入项目。

## 2026-06-17：补充工程向 Agent skills

- 决策：继续在 `agent-skills/` 中新增后端模块、API/数据契约、AI mock provider 三个工程向 skills。
- 原因：LifePilot 后续 P0 任务高度重复地涉及空间权限、REST 合约、Flyway 迁移、前端 API 类型和 AI 草稿确认流程；将这些流程抽成 skills 比引入通用艺术、Artifact、Claude API 或文档格式 skills 更贴合当前阶段。

## 2026-06-18：收敛 Agent 文档权威关系

- 决策：将 `docs/CURRENT_STATE.md` 作为唯一当前状态源，`docs/BACKLOG.md` 作为唯一任务源，`docs/CHANGELOG_AGENT.md` 作为历史记录；`docs/HANDOFF.md` 改为稳定运行手册，`docs/NEXT_CHAT_PROMPT.md` 改为极简入口。
- 原因：多份文档重复记录当前阶段和下一任务会产生漂移，影响新 Agent 接手；收敛权威关系并增加文档检查脚本，可以降低自主开发收尾成本。

## 2026-06-18：按改动类型执行提交前检查

- 决策：所有提交前必须执行 `git status --short` 和 `git diff --check`；涉及代码、配置、依赖、构建、数据库迁移、前端资源或运行行为的改动才强制执行后端测试和前端构建。纯文档改动可以跳过代码验证，但最终回复必须说明跳过项和已执行检查。
- 原因：纯文档提交强制跑全量后端和前端验证成本较高，且收益有限；按改动类型分层验证能保持透明度和效率，同时避免静默跳过必要检查。

## 2026-06-18：提升 Git 规则触发优先级

- 决策：在 `AGENTS.md` 增加 Git 操作硬规则，并在 `docs/AGENT_GIT_RULES.md` 明确触发条件和提交信息自检要求；凡涉及提交、暂存、推送或 amend 的任务，都必须先读取 Git 规则。
- 原因：仅把 Git 规则放在“按任务类型继续阅读”中，容易被 Agent 误判为普通文档任务而跳过，导致英文提交信息或检查流程未触发。
